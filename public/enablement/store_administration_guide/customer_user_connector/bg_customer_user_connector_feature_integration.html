Feature Integration - Customer User Connector

Prerequisites

To prepare your project to work with Customer User Connector:

  1. Require the Customer User Connector modules in your composer by running
    • composer require spryker/customer-user-connector
    • composer require spryker/customer-user-connector-gui
  2. Install the required changes to the data structure by running console propel:diff && console propel:migrate && console propel:model:build. This will generate a migration file, run the migration against your database, and build all new Propel models.
  3. Run vendor/bin/console transfer:generate to generate the new transfer objects.
  4. Make sure the new Zed user interface assets are built by running npm run zed (or antelope build zed for older versions).

Zed Setup

Accessing Connected Users From "Users List" UI

Register UsersTableExpanderPlugin plugin to enable "Assigned Customers" button on "Users List" Zed UI.

Example of users table expander plugin registration:
<?php
                namespace Pyz\Zed\User;

                use Spryker\Zed\CustomerUserConnectorGui\Communication\Plugin\UsersTableExpanderPlugin;
                use Spryker\Zed\User\UserDependencyProvider as SprykerUserDependencyProvider;

                class UserDependencyProvider extends SprykerUserDependencyProvider
                {
                    /**
                     * @return \Spryker\Zed\User\Dependency\Plugin\UsersTableExpanderPluginInterface[]
                     */
                    protected function getUsersTableExtenderPlugins()
                    {
                        return [
                            new UsersTableExpanderPlugin(),
                        ];
                    }
                }

You should now be able to access the Zed API of Customer User Connection management from "Users List" Zed UI.

Displaying Connected User on Zed Customer View UI

To display assigned Zed User in Customer View page, register the CustomerTransferUsernameExpanderPlugin plugin.
<?php
                    namespace Pyz\Zed\Customer;

                    use Spryker\Zed\Customer\CustomerDependencyProvider as SprykerCustomerDependencyProvider;
                    use Spryker\Zed\CustomerUserConnector\Communication\Plugin\CustomerTransferUsernameExpanderPlugin;

                    class CustomerDependencyProvider extends SprykerCustomerDependencyProvider
                    {
                        /**
                         * @return \Spryker\Zed\Customer\Dependency\Plugin\CustomerTransferExpanderPluginInterface[]
                         */
                        protected function getCustomerTransferExpanderPlugins()
                        {
                            return [
                                new CustomerTransferUsernameExpanderPlugin(),
                            ];
                        }
                    }

Usage in Yves

Checking Customer-User Connection

Customer-user connection can be used to restrict access to page.

<?php
            namespace Pyz\Yves\Cms\Controller;

            use Pyz\Yves\Application\Controller\AbstractController;

            class PreviewController extends AbstractController
            {
               /**
                * @return bool
                */
                protected function hasUser()
                {
                   $customer = $this->getFactory()->getCustomerClient()->getCustomer();

                   if ($customer === null || empty($customer->getFkUser())) {
                       return false;
                   }

                   return true;
                }
            }
